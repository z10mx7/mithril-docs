# Table of Contents (TOC) Customization

The Table of Contents (TOC) in this documentation template is defined in the `config/toc.tsx` file. It provides quick reference links on the right side of each documentation page, allowing users to navigate sections efficiently.

## Editing Table of Contents (TOC)

The TOC is structured as an object where each key represents a unique `slug` corresponding to a documentation page. These keys map to an array of TOC sections.

### TOC Structure

Each TOC entry follows this format:

```tsx
'page-slug': [
    {
        title: string; // Section title
        href: string; // Link to the section
        pages?: {
            title: string; // Sub-section title
            href: string; // Link to the subsection
        }[];
    }
]
```

The `page-slug` is derived from the `slug` metadata of the respective documentation page.

### Example TOC Configuration

```tsx
export const TocData = {
  'getting-started/introduction': [
    {
      title: 'Welcome to Pinexio',
      href: '/docs/getting-started/introduction#welcome-to-pinexio',
      pages: [
        {
          title: 'Why Choose Pinexio',
          href: '/docs/getting-started/introduction#why-choose-pinexio',
        },
        {
          title: 'What Makes Pinexio Special',
          href: '/docs/getting-started/introduction#what-makes-pinexio-special',
        },
      ],
    },
  ],
  'components/button': [
    {
      title: 'Preview & Code Snippet',
      href: '/docs/components/button#preview-and-code-snippet',
    },
    {
      title: 'Props',
      href: '/docs/components/button#props',
    },
  ],
};
```

### Adding a New TOC Entry

To add a new TOC entry, insert a new key-value pair in `TocData`:

```tsx
'new-section/page-slug': [
    {
        title: "New Section",
        href: "/docs/new-section#new-section",
    }
]
```

### Removing a TOC Entry

Simply delete the corresponding key from `TocData`.

## Where is the TOC Rendered?

The TOC is used in the `toc.tsx` component, which dynamically retrieves TOC data based on the `slug` value of the active page.

## Summary

- The TOC structure is defined in `config/toc.tsx`.
- Each TOC key corresponds to a unique `slug` for matching documentation pages.
- TOC modifications update the right-side quick reference links.
- The `toc.tsx` component dynamically retrieves and displays TOC data.
- Changes are simple and allow for easy customization.

This guide ensures developers can efficiently modify and maintain the TOC structure for their documentation.
