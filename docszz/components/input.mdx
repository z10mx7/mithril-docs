# Input Component Documentation

The **Input** component is a fully customizable text input field that supports multiple variants, sizes, and states. It's ideal for form elements in modern, theme-aware applications.

## Props

| Prop        | Type                                               | Default      | Description                                                                 |
|-------------|----------------------------------------------------|--------------|-----------------------------------------------------------------------------|
| `variant`   | `'outline' \| 'filled' \| 'none'`                  | `'outline'`  | Controls the input's background and border style.                          |
| `size`      | `'xs' \| 'sm' \| 'md' \| 'lg' \| 'xl'`            | `'sm'`       | Controls the height and padding of the input.                              |
| `fullWidth` | `boolean`                                          | `false`      | If true, the input takes up full width of its container.                   |
| `isError`   | `boolean`                                          | `false`      | Applies red border and ring to indicate error state.                       |
| `isLoading` | `boolean`                                          | `false`      | Displays a spinning loader icon inside the input on the right.             |
| `...props`  | `React.InputHTMLAttributes<HTMLInputElement>`      | â€“            | Inherits all native input attributes.                                      |

## Styling

This input uses Tailwind CSS and `class-variance-authority` to manage style variants, making it easy to scale with design tokens or theming systems.

## Accessibility

Ensure to associate `<label>` components with the `Input` using the `htmlFor` prop for optimal accessibility.
