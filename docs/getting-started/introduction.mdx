# Welcome to Mithril

Mithril is a batteries-included web framework built on top of Fiber v2 with features that inspired from famus frameworks like FastAPI, Django, Express, Nest and Laravel. It brings the developer experience of modern frameworks to the Go ecosystem, providing everything you need to build production-ready web applications.

- **Code Generation**: Generate controllers, models, migrations, CRUD and more with simple commands
- **Hot Reload**: Live reload during development with Air integration

## Key Features

### Core Framework
- **Configuration Management**: Environment-based config with validation
- **Request/Response Validation**: FastAPI-style schema validation
- **Auto-generated Swagger**: OpenAPI 3.0 documentation at `/docs` , Like FastAPI
- **Module System**: Django-style app organization
- **Template Engine**: HTML templates with Go's `html/template`

### Database & Migrations
- **Multi-database Support**: PostgreSQL, Clickhouse, MongoDB, Oracle, MySQL, SQL Server, SQLite, 
- **Migrations**: Up/Down methods with rollback support, Like Django and Laravel
- **Database Seeding**: Populate databases with test data
- **Backup & Restore**: Full database backup and restore functionality, like Django but more advanced 
- **UUID Primary Keys**: Secure, globally unique identifiers by default

### Authentication & Authorization
- **JWT Tokens**: Secure JSON Web Token authentication
- **Session Management**: Database-backed session storage
- **2FA Support**: TOTP authenticator apps
- **Email Verification**: Built-in email verification system
- **Password Reset**: Secure password reset flow
- **RBAC**: Role-based access control with roles and permissions
- **RequestID**: every request has its own unique ID to make easier Debugging user issues even in Production



### File Storage
- **Object Storage**: AWS S3 Integration and MinIO Support, no more Local Storage at all
- **File Operations**: Upload, download, stream, list, and delete files and folders using builtin Filemanagement API
- **Backup/Restore**: Storage backup and restore functionality

### Background Jobs
- **Queue System**: RabbitMQ, Kafka, Memory and Redis support
- **Task Scheduling**: Cron-like scheduling with recurring tasks with monitoring tools for queue
- **Job Retries**: Automatic retry with exponential backoff
- **Failed Job Handling**: Dead letter queue and retry mechanisms

### Real-time Communication
- **WebSocket Support**: Real-time communication with room management
- **gRPC Support**: High-performance RPC communication
- **Broadcasting**: Message broadcasting to multiple clients

### Monitoring & Observability
- **Health Checks**: `/health`, `/livez`, `/readyz` endpoints
- **Prometheus Metrics**: `/pmetrics` endpoint for monitoring
- **OpenTelemetry**: Distributed tracing support
- **Sentry Integration**: Error tracking and monitoring
- **System Monitoring**:  `/metrics` endpoint for CPU, memory, disk, and network monitoring
