# Welcome to Mithril

Mithril is a batteries-included AI-Friendly Golang web framework built on top of Fiber with features that inspired from famus frameworks like FastAPI, Django, Express, Nest and Laravel. It brings the developer experience of modern frameworks to the Go ecosystem, providing everything you need to build production-ready web applications.

## Why Mithril?

### 🚀 **Developer Experience**

- **Amazing CLI**: Global CLI tool (`mithril new project`) + local commands , consider it something similar to django-admin or django manage.py or laravels artisan
- **Code Generation**: Generate CRUD (controllers, migrations and more) from your Models and Dtructs
- **Hot Reload**: Live reload during development with Air integration
- **Dockerized**: Production Ready and Size Optimized Docker Image with builtin Health Checks .
- **Auto Swagger Generator**: Automaticaly Generate and Update Swagger Documents and OpenAPI Specs Using AI .
- **Database Migrations**: Advanced Database Migrations using goose , Database Seeders, Database Backup, Database Restore
- **Database Backups**: Supports Multiplie Backup Versions From Database and Restoring them from List of Your Backups
- **Database Seeders**: Supports Database Seeders
- **Ready to Use Services**: mithril CLI will help you to easily Run and Manage Containers You May Need during development, including Postgres, PgAdmin, Adminer, Redis, RabbitMQ, Kafka, Minio, Rustfs, Postgresus
- **Lightweight Database Management System**: mithril has a unique fast and lightweight dbms thats inspired by adminer and runs on port 5050 with `mithril dbms` command, can be enabled and disabled by `dbms-enable` and `dbms-disable`
- **Mithril Admin**: mithril has a lightweight admin panel that inspired by django admin web and some its features, located at /admin . can be enabled and disabled by `mithril admin-enable` and `mithril admin-disable` commands
- **Encryption Ultilities**: Built-in Commands for Encryption, Hashing and Encoding . including bcrypt, SHA-256, SHA-512, Base64, openssl secret
- **Comprehensive Testing**: Built-in testing utilities and helpers

### 🏗️ **Modern Architecture**

- **Fiber v2**: High-performance HTTP framework built on fasthttp
- **PostgreSQL Integration**: using postgresql and pgx to acheive better performance during database interactions
- **Dependency Injection**: Laravel-style service container
- **Reposity Pattern Ready**: Reposity Pattern is First class citizen here
- **Faster Json Render**: mithril uses sonic for acheiving better Performance during parsing jsons
- **Middleware Stack**: CORS, CSRF, rate limiting, sessions, and more

### 🔐 **Security First**

- **Encryption**: AES Encryption and MistHash support
- **JWT Authentication**: Access and refresh token management
- **RBAC System**: Django Like Role-based access control with permissions
- **CSRF Protection**: Built-in CSRF token validation
- **Rate Limiting**: Per-IP, per-user, and per-route rate limiting
- **Security Headers**: Helmet-style security headers
- **RequestID**: every request has its own unique ID to detect attacker easier
- **CORS**: Supports Cors By default
- **Built-in Captcha**: No Dependency Numeric Captcha, Image and Text
- **Built-in Helpers**: builtin helper functions to check ownership of an entity

### 📊 **Production Ready**

- **Health Checks**: Built-in health check endpoints
- **Monitoring**: Prometheus metrics and OpenTelemetry and Sentry support
- **Logging**: Structured JSON logging with multiple levels
- **Docker Support**: Multi-stage Dockerfile and docker-compose configurations
- **Kubernetes**: Complete K8s deployment manifests
- **Github Actions**: Complete Github Actions for CI/CD and Deployment Notifications
- **GZIP**: Gzip Support by default

## Key Features

### Core Framework

- **Configuration Management**: Environment-based config with validation
- **Request/Response Validation**: FastAPI-style schema validation
- **Auto-generated Swagger**: OpenAPI 3.0 documentation at `/docs` , Like FastAPI but Using AI
- **Module System**: Django-style app organization
- **Template Engine**: HTML templates with using JET engine

### Database & Migrations

- **Goose AutoMigrate**: Automatic schema migration based on model/struct definitions using goose
- **Database Seeding**: Populate database with test/dummy/init data
- **Backup & Restore**: Full database backup and restore functionality, like Django but more advanced
- **UUID Primary Keys**: Secure, globally unique identifiers by default

### Authentication & Authorization

- **JWT Tokens**: Secure JSON Web Token authentication
- **Session Management**: Database-backed session storage
- **2FA Support**: TOTP authenticator apps
- **Email Verification**: Built-in email verification system
- **Password Reset**: Secure password reset flow
- **RBAC**: Role-based access control with roles and permissions
- **RequestID**: every request has its own unique ID to make easier Debugging user issues even in Production

### File Storage

- **Object Storage**: AWS S3 Integration, MinIO/RustFS Support, no more Local Storage at all
- **File Operations**: Upload, download, stream, list, and delete files and folders using builtin File Management API
- **Backup/Restore**: Storage backup and restore functionality on remote/local target using single command

### Background Jobs

- **Queue System**: RabbitMQ, Kafka, Memory and Redis support
- **Task Scheduling**: Cron-like scheduling with recurring tasks with monitoring tools for queue
- **Job Retries**: Automatic retry with exponential backoff
- **Failed Job Handling**: Dead letter queue and retry mechanisms

### Real-time Communication

- **WebSocket Support**: Real-time communication with room management, simulating wocketio by wrapping around ws
- **gRPC Support**: High-performance RPC communication
- **Broadcasting**: Message broadcasting to multiple clients

### Monitoring & Observability

- **Health Checks**: `/health`, `/livez`, `/readyz` endpoints
- **System Monitoring**: `/monitor` endpoint for CPU, memory, disk, and network monitoring
- **OpenTelemetry**: Distributed tracing support
- **Sentry Integration**: Error tracking and monitoring with both sentry and mithrilbounter
- **Prometheus Metrics**: `/pmetrics` endpoint for monitoring !!!
