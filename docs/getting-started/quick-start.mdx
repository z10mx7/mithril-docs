# Quick Start

Get up and running with Mithril in just a few minutes! This guide will walk you through creating your first Mithril application.

## Create Your First Project

### 1. Install Mithril

```bash
# Install the global CLI tool
go install github.com/mithril-framework/mithril/cmd/mithril@latest
```

### 2. Create a New Project

```bash
# Create a new Mithril project
mithril new hello-mithril

# Navigate to the project directory
cd hello-mithril
```

### 3. Install Dependencies

```bash
# Install Go dependencies
go mod tidy
```

### 4. Configure Environment

```bash
# Copy the example environment file
cp .env.example .env

# The default configuration will work for this quick start
```

### 5. Start the Development Server

```bash
# Start the development server
go run . artisan serve
```

### 6. Visit Your Application

Open your browser and visit:
- **Web Application**: http://localhost:3000
- **API Documentation**: http://localhost:3000/docs
- **Health Check**: http://localhost:3000/health

🎉 **Congratulations!** You've created your first Mithril application!

## Your First API Endpoint

Let's create a simple API endpoint to get familiar with Mithril:

### 1. Create a Controller

```bash
# Generate a new controller
go run . artisan make:controller HelloController
```

This creates `app/controllers/hello_controller.go`:

```go
package controllers

import (
    "github.com/gofiber/fiber/v2"
)

type HelloController struct{}

func NewHelloController() *HelloController {
    return &HelloController{}
}

func (c *HelloController) Index(ctx *fiber.Ctx) error {
    return ctx.JSON(fiber.Map{
        "message": "Hello from Mithril!",
        "version": "1.0.0-alpha",
        "timestamp": time.Now().Unix(),
    })
}
```

### 2. Create a Route

Edit `routes/api.go`:

```go
package routes

import (
    "github.com/gofiber/fiber/v2"
    "hello-mithril/app/controllers"
)

func SetupAPIRoutes(app *fiber.App) {
    api := app.Group("/api")
    
    helloController := controllers.NewHelloController()
    
    api.Get("/hello", helloController.Index)
}
```

### 3. Test Your Endpoint

```bash
# Test the endpoint
curl http://localhost:3000/api/hello
```

You should see:

```json
{
    "message": "Hello from Mithril!",
    "version": "1.0.0-alpha",
    "timestamp": 1704067200
}
```

## Your First Database Model

Let's create a simple User model and see Mithril's database features:

### 1. Create a Model

```bash
# Generate a new model
go run . artisan make:model User
```

This creates `app/models/user.go`:

```go
package models

import (
    "time"
    "github.com/google/uuid"
    "gorm.io/gorm"
)

type User struct {
    ID        uuid.UUID `json:"id" gorm:"type:uuid;primary_key;default:gen_random_uuid()"`
    Email     string    `json:"email" gorm:"uniqueIndex;not null"`
    Name      string    `json:"name" gorm:"not null"`
    CreatedAt time.Time `json:"created_at"`
    UpdatedAt time.Time `json:"updated_at"`
    DeletedAt gorm.DeletedAt `json:"deleted_at" gorm:"index"`
}
```

### 2. Register Models for Migration

Mithril uses GORM's AutoMigrate feature. Register your models in the model file:

```go
// app/models/user.go
package models

import (
    "time"
    "github.com/google/uuid"
    "github.com/mithril-framework/mithril/pkg/database/migration"
    "gorm.io/gorm"
)

type User struct {
    ID        uuid.UUID `json:"id" gorm:"type:uuid;primary_key;default:gen_random_uuid()"`
    Email     string    `json:"email" gorm:"uniqueIndex;not null"`
    Name      string    `json:"name" gorm:"not null"`
    CreatedAt time.Time `json:"created_at"`
    UpdatedAt time.Time `json:"updated_at"`
    DeletedAt gorm.DeletedAt `json:"deleted_at" gorm:"index"`
}

// Register the model for AutoMigrate
func init() {
    migration.RegisterModel(&User{})
}
```

**Alternative**: You can also call `db.AutoMigrate()` directly in your application code instead of using the registry.

### 3. Run the Migration

```bash
# Run migrations (GORM AutoMigrate)
go run . artisan migrate
```

GORM AutoMigrate will automatically:
- Create tables if they don't exist
- Add missing columns
- Add missing indexes
- Add missing foreign keys

**Note**: GORM AutoMigrate won't delete unused columns or change column types for safety.

### 4. Create a Seeder

```bash
# Generate a seeder
go run . artisan make:seeder UserSeeder
```

This creates `database/seeders/user_seeder.go`:

```go
package seeders

import (
    "gorm.io/gorm"
    "hello-mithril/app/models"
)

type UserSeeder struct{}

func (s *UserSeeder) Run(db *gorm.DB) error {
    users := []models.User{
        {Email: "admin@example.com", Name: "Admin User"},
        {Email: "user@example.com", Name: "Regular User"},
    }
    
    for _, user := range users {
        db.Create(&user)
    }
    
    return nil
}
```

### 5. Run the Seeder

```bash
# Run seeders
go run . artisan db:seed
```

## Your First Module

Let's create a complete module with CRUD operations:

### 1. Create a Module

```bash
# Generate a complete module
go run . artisan make:module blog --full
```

This creates a complete blog module with:
- Models (Post, Category)
- Controllers (PostController, CategoryController)
- Routes (API and web routes)
- Schemas (Request/response validation)
- Views (HTML templates)

### 2. Register and Run Migrations

Make sure to register any new models in their `init()` functions, then run:

```bash
# Run migrations for the new module
go run . artisan migrate
```

### 3. Test the Module

```bash
# Test the blog API
curl http://localhost:3000/api/posts
curl http://localhost:3000/api/categories
```

## Authentication Setup

Let's add authentication to your application:

### 1. Scaffold Authentication

```bash
# Generate authentication system
go run . artisan make:auth
```

This creates:
- Authentication controllers
- JWT token management
- Login/register routes
- Password reset functionality
- Email verification

### 2. Create a Superuser

```bash
# Create an admin user
go run . artisan createsuperuser
```

### 3. Test Authentication

```bash
# Register a new user
curl -X POST http://localhost:3000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"password123","name":"Test User"}'

# Login
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"password123"}'
```

